🚀 Guía de Despliegue

Requisitos previos

  • Cuenta de GitHub
  • Git instalado
  • Ruby ≥ 3.0 con Bundler
  • Node.js ≥ 18

Instalación local

git clone https://github.com/tu-usuario/tu-usuario.github.io.git
cd tu-usuario.github.io
bundle install
npm install

Ejecución en desarrollo

npm start
# Esto compila Tailwind, inicia Jekyll y live reload

Abre http://localhost:4000 en tu navegador.

Configuración de variables

Edita _config.yml y los archivos JS para reemplazar los placeholders:

  • TU_FORM_ID → tu ID de Formspree (en contact-form.js y newsletter-signup.js)
  • TU_PRODUCTO_CPP → tu ID de Gumroad
  • ca-pub-XXXXXXXXX → tu ID de AdSense (opcional)
  • Enlaces a redes sociales en _includes/footer.html
  • Hashes de integridad SRI en _layouts/default.html y _includes/head.html (generar en srihash.org)

Desplegue automático

Al hacer push a la rama main, GitHub Actions:

  1. Instala Ruby N Node.
  2. Compila Tailwind CSS.
  3. Construye el sitio con Jekyll.
  4. Valida HTML.
  5. Despliega en GitHub Pages.

Asegúrate de que en Settings → Pages la fuente sea GitHub Actions.

Uso de Cloudflare (opcional, recomendado)

Para añadit WAF, rate limiting y cabeceras de seguridad reales:

  1. Registra tu dominio en Cloudflare.
  2. Configura SSL "Full (strict)".
  3. Habilita HSTS.
  4. Añade reglas de rate limiting en Page Rules.
  5. Configura el Cloudflare Worker para rate limiting avanzado.
